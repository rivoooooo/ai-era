# Challenge 多语言支持实施计划

## 需求概述
为 challenges 表添加语言支持，使同一题目能够提供多个语言版本，并通过语言切换功能展示不同语言的内容。

## 现有结构分析

### challenges 表当前字段
| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| category_id | UUID | 外键，关联 categories |
| slug | VARCHAR | URL 标识 (唯一) |
| name | VARCHAR | 挑战名称 |
| description | TEXT | 挑战描述 |
| difficulty | VARCHAR | 难度等级 |
| created_at | TIMESTAMP | 创建时间 |

## 实施方案

### 步骤 1: 修改 schema.ts
- 在 challenges 表中添加 `language` 字段（VARCHAR，默认 'en'）
- 修改 slug 唯一性约束为 (slug, language) 组合唯一
- 添加对应的索引优化查询性能

### 步骤 2: 更新 queries.ts
- 修改 `getChallengeBySlug` 函数支持语言参数
- 修改 `getCategoriesWithChallenges` 函数支持语言过滤
- 新增 `getChallengesByLanguage` 辅助函数
- 新增 `getAvailableLanguages` 函数获取可用语言列表

### 步骤 3: 更新 seed.ts
- 为现有种子数据添加 language 字段
- 示例：添加中文版本的题目

### 步骤 4: 更新 drizzle 配置并生成迁移
- 运行 `bun run db:generate` 生成迁移文件
- 运行 `bun run db:push` 推送到数据库
- 运行 `bun run db:seed` 重新导入种子数据

## 数据库变更 (SQL)

```sql
-- 添加 language 字段
ALTER TABLE challenges ADD COLUMN language VARCHAR(10) NOT NULL DEFAULT 'en';

-- 删除原有的 slug 唯一约束
ALTER TABLE challenges DROP CONSTRAINT challenges_slug_key;

-- 添加 (slug, language) 组合唯一约束
ALTER TABLE challenges ADD CONSTRAINT challenges_slug_language_unique UNIQUE (slug, language);

-- 添加索引优化查询
CREATE INDEX idx_challenges_language ON challenges(language);
CREATE INDEX idx_challenges_slug_language ON challenges(slug, language);
```

## TypeScript 类型更新

```typescript
// Challenge 表新增字段
language: text('language').notNull().default('en'),

// 唯一约束
slugLanguageUnique: unique('challenges_slug_language_unique').on(table.slug, table.language),
```

## 语言切换功能设计

- 前端可通过 URL 参数或查询参数切换语言（如 `?lang=zh`）
- API 查询支持按 language 过滤
- 默认语言为 'en'
- 支持的语言：'en', 'zh', 'ja', 'es' 等

## 预期效果
- 同一题目可以创建多个语言版本
- URL 示例：`/challenge/javascript-arrays` (英文)、`/challenge/javascript-arrays-zh` (中文)
- 语言切换时显示对应版本的题目名称和描述
